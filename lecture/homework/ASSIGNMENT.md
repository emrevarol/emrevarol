# PR Review Assignment: Review Like a Professional Engineer

## Your Task

Review the codebase in this repository as if you were a senior engineer reviewing a pull request.

## Requirements

Your submission must include:

1. **At least 10 review comments** total
2. **At least 3 naming-related comments**
3. **At least 2 testing-related comments**
4. **At least 2 comments about design or function responsibility**
5. **At least 1 comment about repository quality**
6. **At least 1 refactoring suggestion**

## Submission Format

Submit your review as a text file or markdown file using this structure:

```
File: src/orders.py
Line/Area: calc function
Comment: The function name is too generic. Could we rename it to calculate_order_total for clarity?

File: tests/test_orders.py
Line/Area: test_calc
Comment: This test only checks that the result is greater than zero. Could we assert the exact expected value instead?
```

## Evaluation Criteria

- Quality of observations (not just quantity)
- Professional tone in comments
- Specificity of suggestions
- Coverage across naming, testing, design, and repository quality
